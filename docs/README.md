# Requisitos funcionales

**1.	Gestión de usuarios**
-	Registrar nuevos usuarios con nombre, documento y tipo (estudiante, docente, administrativo, externo).
-	Consultar y actualizar la información de los usuarios registrados.

**2.	Gestión de películas y cartelera**
-	Mostrar la lista de películas disponibles para el fin de semana (con horarios, duración y clasificación).

**3.	Gestión de reservas y asientos**
-	Permitir seleccionar asientos específicos en un mapa interactivo de la sala.
-	Prevenir reservas duplicadas y notificar si el asiento ya está ocupado.
-	Cancelar reservas y liberar automáticamente los asientos.

**4.	Procesos de facturación y cobro**
-	Calcular el valor de la entrada según el tipo de usuario.
-	Generar facturas digitales con número único de comprobante.
-	Guardar las facturas en un archivo (CSV o TXT) para control administrativo.

**5.	Reportes y estadísticas**
-	Generar reportes de ingresos diarios y semanales.
-	Mostrar el número de reservas por película, usuario y tipo de público.
-	Indicar la ocupación promedio de la sala en cada función.

**6.	Interacción en consola**
-	Ofrecer un menú principal amigable con opciones claras (registrar, reservar, cancelar, facturar, reportes).
-	Confirmar cada acción con mensajes claros y personalizados al usuario.

# Requisitos no funcionales

**1.	Accesibilidad**
-	El usuario debe contar con acceso a internet.
-	Disponer de un computador, portátil o dispositivo móvil con navegador actualizado.
-	Tener un correo institucional o cuenta de Google válida para acceder.

**2.	Seguridad**
-	La información de los usuarios debe almacenarse de manera confidencial.
-	El sistema debe validar credenciales antes de permitir el acceso.

**3.	Disponibilidad**
-	El sistema debe estar disponible en todo momento dentro de los horarios de reserva.
-	Debe recuperarse rápidamente en caso de fallos.

**4.	Fiabilidad**
-	El sistema debe garantizar que las reservas realizadas no se pierdan.
-	Debe registrar correctamente todas las transacciones.

**5.	Rendimiento**
-	La plataforma debe responder en menos de 3 segundos en condiciones normales.
-	Debe soportar múltiples usuarios realizando reservas de forma simultánea.

**6.	Usabilidad**
-	La interfaz debe ser intuitiva y fácil de usar para la comunidad universitaria.
-	El sistema debe adaptarse a distintos dispositivos (computadores, tablets, móviles).
